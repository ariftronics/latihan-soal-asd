1. Analisis Kompleksitas (Big-O)

Soal 1.1Mudah

Tentukan Big-O:

for (int i = 0; i < n; i++) {
    printf("%d", i);
}
PembahasanLoop `n` kali dengan operasi konstan → O(n) (linear).
Soal 1.2Sedang
for (int i = 0; i < n; i++)
    for (int j = 0; j < n; j++)
        sum += a[i][j];
PembahasanDua loop bersarang → n × n = O(n²).
Soal 1.3Sulit
for (int i = 1; i < n; i = i * 2) {
    printf("%d", i);
}
Pembahasan`i` digandakan tiap iterasi (1,2,4,8…) → log₂(n) iterasi → O(log n).

2. Array

Soal 2.1Mudah

Array A alamat awal 1000, tiap int 4 byte. Alamat A[3]?

Pembahasan1000 + (3 × 4) = 1012.
Soal 2.2Sedang

Keunggulan & kelemahan array vs linked list?

PembahasanUnggul: akses acak O(1), hemat memori. Lemah: ukuran statis; sisip/hapus di tengah O(n).

3. Stack & Queue

Soal 3.1Mudah

Stack (bawah→atas) 2,5,8. Setelah pop() lalu push(10)?

Pembahasanpop() buang 8 (LIFO) → 2,5; push(10) → 2, 5, 10.
Soal 3.2Sedang

enqueue(A), enqueue(B), dequeue(), enqueue(C), dequeue(). Sisa?

PembahasanFIFO: keluar A lalu B → sisa [C].
Soal 3.3Sulit

Ubah infix A + B * C ke postfix & peran stack.

PembahasanPostfix A B C * +. Stack simpan operator per prioritas; * > + jadi B C * dulu.

4. Linked List

Soal 4.1Mudah

Beda singly vs doubly linked list?

PembahasanSingly: 1 pointer (next), hanya maju. Doubly: prev & next, maju-mundur tapi butuh memori lebih.
Soal 4.2Sedang

head→10→20→30. Sisipkan 15 antara 10 dan 20.

Pembahasan1) buat newNode(15); 2) newNode->next = node(10)->next; 3) node(10)->next = newNode. Hasil 10→15→20→30, O(1).
Soal 4.3Sulit

Kenapa cari di linked list O(n) & kenapa cocok untuk stack/queue?

PembahasanTak ada index → telusuri dari head O(n). Cocok karena dinamis & push/pop ujung O(1).

5. Sorting

Soal 5.1Mudah

Bubble sort [5,2,9,1] menaik, tiap pass?

PembahasanPass1 [2,5,1,9] → Pass2 [2,1,5,9] → Pass3 [1,2,5,9].
Soal 5.2Sedang

Ide Merge Sort & kenapa O(n log n)?

PembahasanDivide & Conquer: bagi dua sampai 1 elemen lalu gabung terurut. log n tingkat × n operasi → O(n log n).
Soal 5.3Sulit

Kapan Quick Sort worst case O(n²) & cara hindari?

PembahasanPivot buruk (data terurut, pivot ujung) → partisi 1 vs n-1. Hindari: pivot acak / median-of-three.

6. Searching

Soal 6.1Mudah

Perbandingan linear search terburuk pada 8 elemen?

Pembahasan8 perbandingan → O(n).
Soal 6.2Sedang

Binary search [1,3,5,7,9,11,13] cari 11.

Pembahasanmid idx3=7<11 → kanan; mid idx5=11 ✓ (2 langkah, O(log n)).
Soal 6.3Sulit

Kenapa binary search butuh data terurut?

PembahasanMembuang separuh ruang per langkah berdasar urutan. Acak & sekali → linear O(n); berkali-kali → urutkan dulu lalu binary.
